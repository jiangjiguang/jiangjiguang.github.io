博客环境搭建

发表于 2019-12-21 更新于 2021-06-20 分类于 blog 阅读次数：

安装Git

自行安装

安装完成后，还需要最后一步设置，在命令行输入

$ git config –global user.name “name”

$ git config –global user.email “email”

安装Node.js

下载地址：https://nodejs.org/en/

部署Hexo

Hexo官网：https://hexo.io/zh-cn/docs/writing

安装Hexo

打开终端，运行以下命令完成安装

npm install -g hexo-cli

安装Hexo的Git部署插件hexo-deployer-git：https://github.com/hexojs/hexo-deployer-git

打开终端运行以下命令：

npm install hexo-deployer-git -save

部署Hexo

打开终端

hexo init blog //会在当前目录新建blog目录，用于存放博客内容

cd blog

npm install

输入hexo s 启动博客

打开浏览器输入 http://localhost:4000/ 即可访问

主题设置

https://theme-next.iissnan.com/getting-started.html

托管博客

托管到github

首先需要在github建立两个repository：

  • 一个用来放置博客文件，用来备份博客的原始内容，以及用来在多台电脑上编辑和更新博客

  • 一个用来放置博客被发布的生成好的内容，该repository的名字必须为 你的github用户名.github.io

回到博客根目录：

找到文件 _config.yml 在deploy处增加三行：

deploy:

type: git

repository: 你的仓库地址

branch: master

生成站点文件并推送到远程库

hexo clean && hexo deploy

当执行 hexo deploy 时，Hexo 会将 public 目录中的文件和目录推送至 _config.yml 中指定的远端仓库和分支中，并且完全覆盖该分支下的已有内容。

打开网站，可以看到博客内容：

https://你的仓库名称

仓库1的.gitignore配置

$ cat .gitignore

.DS_Store

Thumbs.db

db.json

*.log

node_modules/

public/

.deploy*/

绑定域名

有没有觉得使用自己的域名会更酷？

  • 首先需要购买域名：https://sg.godaddy.com/zh

在source目录下新建CNAME文件，写入域名，如：jiangjiguang.me

最后执行：hexo deploy

  • 在github中设置域名：

git的repository，点击Settings 往下滑到 GitHub Pages 点击 Change theme （不设置这一步骤会报404）

添加域名解析：

ping 你的仓库名称

得到一个ip：185.199.109.153

  • 配置DSN：

DNS服务商选的dnspod：https://support.dnspod.cn/

配置文档：

https://support.dnspod.cn/Kb/showarticle/tsid/177/

https://support.dnspod.cn/Kb/showarticle/tsid/30/

  • 最后再到godaddy修改域名服务器：

f1g1ns1.dnspod.net

f1g1ns2.dnspod.net
