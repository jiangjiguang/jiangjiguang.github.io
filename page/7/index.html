flink-作业提交

发表于 2020-01-14 更新于 2020-06-22 分类于 flink 阅读次数：
参数 参数全名 作用
-ys –yarnslots 每个taskManager的slot数量
-ytm –yarntaskManagerMemory 每个taskManager Container的内存量（MB）
-yn –yarncontainer 分配的YARN container的数量（等于taskManager）
-yjm –yarnjobManagerMemory JobManager Container的内存（MB）
-p –parallelism 作业的并行度，可以覆盖配置文件中的并行度
该参数覆盖flink配置文件的中并行度
可以在代码中设置并行度覆盖该参数
-s –fromSavepoint 用来恢复作业的 savepoint路径
-ynm –yarnname 设置作业提交到yarn上的任务的名称
-c –class 设置jar的入口类，也可以在jar中配置入口类

export和source

发表于 2020-01-14 更新于 2020-06-22 分类于 linux 阅读次数：

export作用：设置或显示环境变量。

语法：export [-fnp][变量名称]=[变量设置值]

参数：

  -f 代表[变量名称]中为函数名称。

  -n 删除指定的变量。变量实际上并未删除，只是不会输出到后续指令的执行环境中。

  -p 列出所有的shell赋予程序的环境变量。

  用户登录到linux终端后，系统将启动一个用户shell。在这个shell中，可以使用或者声明变量，也可以创建并运行 shell脚本程序。运行shell脚本程序时，系统将创建一个子shell。子shell可以使用父shell的所有环境变量。

  在子 shell中定义的变量只在该子shell内有效。如果在一个shell脚本程序中定义了一个变量，当该脚本程序运行时，这个定义的变量只是该脚本程序内的一个局部变量，其他的shell不能引用它，要使某个变量的值可以在其他shell中被使用，可以使用export命令对已定义的变量进行输出。

  简单来讲，export的作用就是设置环境变量，从而可以在本shell或者子shell中使用该变量，但是不能在父shell中使用。

  如果使用source执行子脚本设置环境变量，那么该环境变量也可以在其父环境（当前环境）下使用

结论：

  1.bash命令执行脚本时是在一个子shell环境运行的，脚本执行完后该子shell自动退出

  2.export定义的环境变量才会被复制到子 shell中

  3.不用export定义的变量只对该shell有效，对子shell也是无效的

  4.使用source执行脚本是在当前环境运行的，所以脚本中的环境变量也对当前shell有效

博客环境搭建

发表于 2019-12-21 更新于 2020-06-22 分类于 blog 阅读次数：

安装Git

自行安装

安装完成后，还需要最后一步设置，在命令行输入

$ git config –global user.name “name”

$ git config –global user.email “email”

安装Node.js

下载地址：https://nodejs.org/en/

部署Hexo

Hexo官网：https://hexo.io/zh-cn/docs/writing

安装Hexo

打开终端，运行以下命令完成安装

npm install -g hexo-cli

安装Hexo的Git部署插件hexo-deployer-git：https://github.com/hexojs/hexo-deployer-git

打开终端运行以下命令：

npm install hexo-deployer-git -save

部署Hexo

打开终端

hexo init blog //会在当前目录新建blog目录，用于存放博客内容

cd blog

npm install

输入hexo s 启动博客

打开浏览器输入 http://localhost:4000/ 即可访问

主题设置

https://theme-next.iissnan.com/getting-started.html

托管博客

托管到github

首先需要在github建立两个repository：

  • 一个用来放置博客文件，用来备份博客的原始内容，以及用来在多台电脑上编辑和更新博客

  • 一个用来放置博客被发布的生成好的内容，该repository的名字必须为 你的github用户名.github.io

回到博客根目录：

找到文件 _config.yml 在deploy处增加三行：

deploy:

type: git

repository: 你的仓库地址

branch: master

生成站点文件并推送到远程库

hexo clean && hexo deploy

当执行 hexo deploy 时，Hexo 会将 public 目录中的文件和目录推送至 _config.yml 中指定的远端仓库和分支中，并且完全覆盖该分支下的已有内容。

打开网站，可以看到博客内容：

https://你的仓库名称

仓库1的.gitignore配置

$ cat .gitignore

.DS_Store

Thumbs.db

db.json

*.log

node_modules/

public/

.deploy*/

绑定域名

有没有觉得使用自己的域名会更酷？

  • 首先需要购买域名：https://sg.godaddy.com/zh

在source目录下新建CNAME文件，写入域名，如：jiangjiguang.me

最后执行：hexo deploy

  • 在github中设置域名：

git的repository，点击Settings 往下滑到 GitHub Pages 点击 Change theme （不设置这一步骤会报404）

添加域名解析：

ping 你的仓库名称

得到一个ip：185.199.109.153

  • 配置DSN：

DNS服务商选的dnspod：https://support.dnspod.cn/

配置文档：

https://support.dnspod.cn/Kb/showarticle/tsid/177/

https://support.dnspod.cn/Kb/showarticle/tsid/30/

  • 最后再到godaddy修改域名服务器：

f1g1ns1.dnspod.net

f1g1ns2.dnspod.net
