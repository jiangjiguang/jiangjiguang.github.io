HDFS架构简介

发表于 2020-06-23 分类于 hadoop 阅读次数：

HDFS的优点和缺点

优点

  • 高容错性

    多副本

    副本丢失，可以自动恢复

  • 适合处理大数据

    能够处理PB级别的数据

    能够处理百万以上的文件数量

  • 可构建的廉价的机器上

缺点

  • 不适合低延迟数据访问
  • 无法高效地对大量小文件进行存储
  • 不支持并发写入，文件随机修改

HDFS的架构

NameNode

  • 管理HDFS的名称空间
  • 配置副本策略
  • 管理数据块（Block）映射信息
  • 处理客户端读写请求

DataNode

NameNode下达命令，DataNode负责执行实际的操作

  • 存储实际的数据块
  • 执行数据块的读写操作

Client客户端

  • 文件切分 文件上传到HDFS的时候，Client将文件切分成一个个的Block，然后进行上传
  • 与NameNode交互，获取文件的位置信息
  • 与DataNode交互，读取或者写入数据
  • 通过命令管理HDFS，比如NameNode格式化等
  • 通过命令访问HDFS，比如对HDFS进行增删查改等操作

Secondary NameNode

不是NameNode的热备，当NameNode挂掉时并不能替换NameNode

  • 辅助NameNode，分担其工作量，比如定期合并fsimage和edits文件，并推送给NameNode

  • 辅助恢复NameNode
